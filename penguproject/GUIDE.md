# 졸업 프로젝트 가이드 (초보자용)

이 문서는 프로그래밍을 전혀 몰라도 이 프로젝트가 어떻게 만들어지고 있는지,
그리고 여러분이 직접 해야 할 일이 무엇인지 따라올 수 있도록 만든 문서입니다.
Claude가 코드를 작성/수정할 때마다 이 문서의 "진행 로그"를 갱신합니다.

---

## 1. 우리가 만드는 것

**요양 보호 앱**: 어르신(환자)의 스마트폰이 낙상(넘어짐)을 자동으로 감지하고,
증상 관련 말/글을 분석해서 보호자에게 이메일/알림으로 리포트를 보내주는 앱.

- 환자용 화면: 백그라운드에서 낙상 감지, 증상 입력
- 보호자용 화면: 알림 확인, 리포트 확인
- 서버: 낙상/증상 이벤트를 받아서 AI로 리포트를 만들고 알림을 보냄
- 배포: 원스토어(One Store)에 앱 등록

## 2. 꼭 알아야 할 용어 (딱 이만큼만)

| 용어 | 쉬운 설명 |
|---|---|
| 저장소(Repository, repo) | 코드를 모아두는 폴더 + 변경 이력. 이 프로젝트 전체가 하나의 저장소 |
| 브랜치(branch) | 저장소 안에서 작업하는 "버전 갈래". 지금은 `claude/stoic-curie-nix4di`라는 브랜치에서 작업 중 |
| 커밋(commit) | 변경 사항을 저장하는 "저장 지점" (게임의 세이브 포인트와 비슷) |
| 푸시(push) | 내 컴퓨터(혹은 이 세션)의 커밋을 GitHub(원격 저장소)로 올리는 것 |
| 백엔드(backend) | 눈에 안 보이는 서버 쪽 프로그램. 데이터 저장, AI 호출, 알림 발송 담당 |
| 프론트엔드/앱(mobile) | 사용자가 직접 만지는 화면. 여기서는 Flutter로 만든 Android 앱 |
| API | 앱과 서버가 대화하는 규칙. "앱이 서버에 이런 요청을 보내면, 서버가 이런 응답을 준다" |
| 패키지/라이브러리 | 남이 미리 만들어둔 코드 뭉치. 가져다 쓰면 처음부터 안 만들어도 됨 |

## 4. 여러분이 실제로 해야 할 일 (Claude가 못 하는 것)

전부 대신 해줍니다. 하지만 아래 몇 가지는 여러분의 컴퓨터/계정에서 직접 해야 합니다:

1. **본인 컴퓨터에 Flutter 설치 + 실제 폰에서 앱 실행해보기** (나중에 안내)
2. **Oracle Cloud 계정 생성 및 무료 서버 발급** (본인 명의 가입 필요, 나중에 안내)

## 6. 낙상 감지, 어떻게 동작하나요?

처음 계획은 TFLite(딥러닝 모델)를 쓰는 거였지만, 학습 데이터 수집·모델 학습까지 하려면
시간이 많이 들어서, 먼저 **임계값(threshold) 기반 알고리즘**으로 동작하는 버전을 만들었습니다.
이 방식도 실제 낙상 감지 논문에서 널리 쓰이는 방식이라 졸업작품 발표에서 충분히 설명 가능합니다.

원리:
1. 폰이 "자유낙하" 상태가 되면(중력만 남아서 가속도 크기가 순간적으로 확 작아짐) 낙상 시작으로 의심
2. 그 직후 2초 안에 "충격"(가속도 크기가 확 커짐, 바닥에 부딪히는 느낌)이 감지되면 → 낙상으로 확정

관련 코드:
- `mobile/lib/models/sensor_sample.dart`: 센서 값 하나 (x, y, z, 시각)
- `mobile/lib/services/fall_detector.dart`: 위 알고리즘 (순수 로직, 테스트 가능)
- `mobile/lib/screens/monitor_screen.dart`: 실제 센서를 읽어서 화면에 보여주는 화면
- `mobile/test/fall_detector_test.dart`: 알고리즘이 제대로 동작하는지 확인하는 자동 테스트

**나중에 개선 가능한 부분** (지금 당장 안 해도 되는 것): 정확도를 더 높이고 싶으면 TFLite
머신러닝 모델로 업그레이드할 수 있습니다. 심사 때 "1차는 임계값 알고리즘으로 구현했고,
추가 개선 방향으로 ML 모델을 고려했다"고 설명하면 오히려 더 좋은 인상을 줄 수 있어요.
